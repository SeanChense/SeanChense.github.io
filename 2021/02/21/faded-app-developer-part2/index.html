没想到之前这篇劝退客户端 曝光率高，不少参与话题讨论不管是支持还是反对都或多或少提到我这篇文章，也没什么规划就随便再谢谢感想。这次就不往 v2ex 上发了，那里涌入了太多水平一般缺乏行业洞见的小年轻，没有办法跟他们进行深层次讨论。不少人还停留在哪个语言好，iOS 还是安卓强这样的肤浅论题上。从历次收集简历的结果来看，几乎拿不到背景过硬的简历。无意攻击 v2ex，相反地我认为一个社区到了这种“不堪”的精英稀释程度反映了用户群庞大，庞大到了它就是整个社会的一个子集——一种比较完美的采样。精英社区没法儿挣钱，嘈杂、不堪才能挣钱。

互喷攻击是网络的固有属性，网络赋予了每个人均等的发言权力。在网络上文盲也能对一个院士评头论足说他能力不行，这跟现实世界完全反过来。不管你承不承认，现实世界的每个人发言权是不均等的。

0x0

打工打不了一辈子，打工也不能致富只能温饱。这话是说给普通人听的，觉得自己是牛人的绕道。

在企业里打工，工资越高工作内容越多，工作时间越长。看着好像是能力提高工资待遇增长了，但还是拿了一部分别的东西来交换，例如陪伴家人的时间、健康的身体。打工是一个死局，要么累死要么穷死。在哪里打工也是个问题，在一线城市打工工资待遇高。不少一线工头工资待遇高于一线工人，就算是北京这样的高房价地区工头也能找个还够得着的地方上车买房定居。但是房子小、老、偏，随即而来的是长距离的通勤和高工资下的高压力以及超长工作时间。这是一线城市对工头的“上车陷阱”，还有另外一种陷阱是对一线工人的——“一线城市的高收入陷阱”。

0x1

回过来再说具体工种，客户端就放到前端大体系下了。前端体系相比于后端有个因素对工人长期规划影响巨大，即业务属性加成。

传统行业做一辈子熟能生巧，后端虽然也有更新换代但是业务属性有加成。智能客服、交易线、收账、搜索等等，每个领域做三年成为熟手，成为很懂这个业务方向的人。对于前端来说，画交易里的列表跟搜索里的列表没有区别。业务本身对前端来说是透明的，任何一个前端都能在快速的时间内在不同的领域能学会怎么写业务。但是后端不行，跨业务面试难度远高于前端。

更直白一点，前端的业务跟后端业务完全是两个级别的东西，虽然他们都叫业务。

0x2

客户端究竟还有什么想象空间？从微信小程序开始是一个巨大变革，资本机器发现可以把很大一部分基建抽象起来让客户端和前端共用节省成本。客户端本身就被掏空了一大块，只剩下写引擎的和别的操作系统贴近的网络、存储（持久、非持久）、APM、埋点。多媒体自成一体，在这里面卷是卷不赢学过 CV、图形处理的人；搞视频，h265/h264 编码也是有人专门学，卷不过；搞语音，大部分人傅里叶都搞不懂还是别跟学过信号处理的人硬拼了。去搞音视频也可以，但只能说是个使用音视频 API 的熟手，正面卷信号处理、图像识别、图形学的时候以卵击石。多媒体里还有搞特效的，看得出来门槛貌似更高。

不少人在讨论客户端的时候会说转音视频，我只能说音视频不是银弹。

0x3

其实这些缺点都不是问题，成年人只看钱，小孩子才分对错。但市场萎缩之下，任何缺点都快速放大。怪只怪资本机器里有人搞出了取代他人的自动化工具——RN/小程序等等，不过这是机器程序的本质。一边掠夺了传统工人的工作另一边是创造了新的商业、就业换来的高工资。

之前讲掠夺他人工作的时候有人动辄就是开倒车，阻碍科技进步的帽子扣过来。更有甚者，珍妮纺织机出来的时候纺织工也是这么想的。你可拉倒吧，好好高考上个本科别读专科才是不阻碍科技进步。

提高生产力是资本的内在动力，不以哪个纺织女工支持不支持为转移。不少搞互联网的人自以为是目空一切，甚至有人看不起银行业乃至整个金融体系，觉得自己就代表了最先进的生产力。互联网的红火让太多人昏了头。
